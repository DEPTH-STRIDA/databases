лаба-0: txt+grep
Выбрать жизненную тему (предметную область). В рамках этой области сделать базу 
данных 20-30 строчек ( больше можно, меньше нет).Таблица 3-4 колонки. Одна 
произвольная,одна пускает перечисление категории. (Например, красный, желтый, 
зеленый и т.д.) Одна-две численные. В текстовом файле в структурированном виде.
Данные можно с интернета, если придумываете сами, то должны быть реалистичные.
Только в текстовом файле. Сделать 2-3 запроса, отвечающих на вопросы по базе 
данных, извлекающим каких-то данных. Используя утилиту grep Линукса. Можно через
онлайн греп. Вставить текст, отфильтровать. Можно запустить и показать в классе.
Файл не удалять, потом с ним будем работать.

Выбор темы:

Выберите жизненную тему или предметную область для вашей базы данных.


Создание структуры базы данных:

Определите 3-4 колонки для вашей таблицы.
Убедитесь, что у вас есть:
а) Одна произвольная колонка
б) Одна колонка с категориями (например, цвета)
в) Одна или две числовые колонки


Сбор данных:

Соберите или придумайте 20-30 строк данных (можно больше, но не меньше).
Если придумываете сами, убедитесь, что данные реалистичны.
Можно использовать данные из интернета.


Создание файла:

Создайте текстовый файл (например, с расширением .txt).
Внесите данные в файл в структурированном виде (например, разделяя колонки
табуляцией или вертикальной чертой |).


Подготовка запросов:

Придумайте 2-3 вопроса, на которые можно ответить, анализируя вашу базу данных.


Использование grep:

Для каждого вопроса создайте команду grep, которая извлечет нужные данные.
Вы можете использовать онлайн-сервис grep или терминал Linux.


Тестирование:

Запустите ваши grep-команды и убедитесь, что они дают ожидаемые результаты.


Документация:

Запишите ваши grep-команды и их результаты.


Сохранение:

Сохраните ваш текстовый файл с данными. Не удаляйте его, так как он может понадобиться для будущих заданий.


Подготовка к презентации:

Будьте готовы продемонстрировать вашу базу данных и grep-запросы в классе.
